# Diretiva: Implementar Feature

## Objetivo
Implementar uma nova feature de forma estruturada e completa.

## Entrada
- Requisitos da feature (do usuário)
- Contexto do projeto (explorado previamente)

## Ferramentas/Scripts
- Agentes: senior-software-engineer, fullstack-dev-specialist
- MCPs: Supabase, Playwright

## Processo (Passo a Passo)

### Passo 1: Análise
- Entender o problema
- Verificar memória para soluções similares
- Identificar dependências

### Passo 2: Planejamento
- Usar agente Plan para definir abordagem
- Dividir em subtarefas
- Definir ordem de execução

### Passo 3: Implementação
- Usar senior-software-engineer para código backend
- Usar fullstack-dev-specialist para código frontend
- Aplicar padrões do projeto

### Passo 4: Testes
- Criar testes unitários
- Criar testes de integração
- Validar com Playwright se necessário

### Passo 5: Documentação
- Atualizar README se necessário
- Adicionar comentários em código complexo
- Salvar solução na memória

### Passo 6: Revisão
- Code review próprio
- Verificar checklist de qualidade

## Saída
- Feature implementada e testada
- Código documentado
- Solução salva na memória

## Edge Cases
- Feature depende de outra → implementar dependência primeiro
- Dados inválidos → validar entrada
- Erro de API → tratar gracefully
